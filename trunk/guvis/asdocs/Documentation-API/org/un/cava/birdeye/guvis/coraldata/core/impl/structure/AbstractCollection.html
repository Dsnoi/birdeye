findAccessorByElement()method 
public function findAccessorByElement(element:Object):IAccessor

Search a collection for an accessor with a particular element and return that accessor. Note that in the case of elements that are Strings and Numbers, more than more accessor could hold the same element. It is up to the implementing collection on how to handle this scenario. Most likely the first instance holding the element would be returned.

element:Object — The element to search for.

remove()method 
public function remove(a:IAccessor, caller:Function = null):Object

Remove an accessor from this collection. This and accessor.destroy() are the only means of removing an accessor from a collection. Subinterfaces may define numerous retrieval methods that can be used to retrieve an accessor at a particular index, which then can be removed.

This method is tightly coupled to the accessor.destroy() method. Therefore, implementing classes must define the following:

if (caller != a.destroy) a.destroy(arguments.callee);

This ensures that if collection.remove(accessor) is called it notifies accessor.destroy() and vice versa. Any class specific code needed to remove an accessor from a particular implementation may also be added.

Parameters
a:IAccessor — The accessor to remove
 
caller:Function (default = null) — The function that called this function, this is used for inter-method communication, so leave it null

Returns
Object — The element stored at the accessor that is being removed
